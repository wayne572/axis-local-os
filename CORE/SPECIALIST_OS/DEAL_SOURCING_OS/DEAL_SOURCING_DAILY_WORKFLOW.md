# DAILY_SOURCING_WORKFLOW.md

## 1. Status
Active - Deal Sourcing Execution File 6 of 6

---

## 2. Purpose
Define the daily routine that ties the Deal Sourcing OS together.

This file turns the system into a habit.

---

## 3. Core Principle

Consistent small action beats sporadic big action.

Sourcing is a daily practice, not a project.

---

## 4. Daily Time Block

Default: 60 minutes per working day.

Split:

- 10 mins - review tracker + next actions
- 15 mins - identify new leads (ICP-matched)
- 25 mins - outreach (new + follow-ups)
- 10 mins - log activity, set tomorrow's next actions

Adjust to capacity. Default beats nothing.

---

## 5. Daily Sequence

1. Open the tracker.
2. Action every lead with a "next action due today."
3. Add new leads to the list (10 verified).
4. Send outreach + follow-ups per Outreach Playbook.
5. Update statuses.
6. Set tomorrow's next actions.
7. Close.

If a step has no work, skip it. Do not invent activity.

---

## 6. Weekly Review

Once a week, 30 minutes:

- count messages sent
- count replies received
- count meetings booked
- count qualified leads moved to Sales OS
- review what worked / what didn't
- adjust messaging or filters if needed

Memory layer captures what worked.

---

## 7. Monthly Reset

Once a month:

- prune the tracker (close cold leads)
- run retention review (6-month rule)
- refresh the ICP based on real data
- audit compliance records (lawful basis, opt-outs)

---

## 8. KPIs (Light)

Default daily / weekly markers:

- 10 verified leads added per day
- 20 outreach actions per day (new + follow-up)
- reply rate target: 8-15%
- 1-3 qualified handoffs per week

These are starting points. The user adjusts based on real data.

---

## 9. DCoS Roles

- ChatGPT DCoS builds the workflow template and time blocks.
- Claude DCoS reviews for over-engineering and unrealistic targets.
- Claude runs the daily check-in and presents the next action clearly.

---

## 10. Failure Signals

The workflow has failed if:

- the user skips multiple days
- the tracker stops being updated
- outreach becomes generic again
- KPIs are not reviewed weekly

Recovery: simplify, not expand. Cut the workflow before adding to it.

---

## 11. End State

A simple daily routine the user actually follows.

Sourcing becomes invisible - just the next action, every day, until pipeline is full.
