# MARKETING_INBOUND_HANDOFF.md

## 1. Status
Active - Execution Sub-File

---

## 2. Purpose
Define how inbound interest generated by Marketing OS hands off to Deal Sourcing OS qualification.

This closes the gap between attention and pipeline.

---

## 3. Core Principle

Inbound is warmer than outbound - but still needs structure.

Inbound replies must enter the same qualification flow as outbound, just at a more advanced stage.

---

## 4. System Rules

- every inbound reply is logged
- every inbound reply is checked against the ICP
- inbound is not auto-qualified - it still goes through the filter
- inbound counts as "Replied" status from day one (skips the cold outreach stage)

---

## 5. Execution Steps

1. Inbound message arrives (DM, email, comment, form).
2. Add to tracker - status: "Replied - Inbound."
3. Check ICP match.
4. If match -> move into Deal Sourcing Qualification Filter at Stage 2 (Reply Quality).
5. If no match -> reply politely and close.
6. If qualified -> Claude suggests Sales OS activation.

---

## 6. Output Requirement

Every inbound must end with:

- a status in the tracker
- a clear next action
- a compliance flag if relevant (AML, B2C, special category)

---

## 7. Failure Conditions

- inbound is replied to without ICP check
- excitement bypasses qualification
- compliance is skipped because the lead "came to us"

---

## 8. End State

Inbound and outbound feed the same disciplined pipeline.

Marketing creates attention. Deal Sourcing qualifies it. Sales converts it. Wayne works directly with Claude.
