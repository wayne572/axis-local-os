# CLIENT_DELIVERY_PROGRESS_TRACKING.md

## 1. Status
Active - Execution Sub-File

---

## 2. Purpose
Track delivery progress clearly.

---

## 3. Core Principle

If it's not tracked, it's not controlled.

---

## 4. System Rules

- track all tasks
- update status regularly
- identify blockers early

---

## 5. Execution Steps

- list all tasks
- mark progress (not started / in progress / complete)
- update daily or regularly
- flag blockers

---

## 6. Output Requirement

Progress must be:

- visible
- accurate
- up to date

---

## 7. Failure Conditions

- missing updates
- unclear progress
- hidden issues

---

## 8. End State

Delivery progress is always clear.
