# CLIENT_DELIVERY_ONBOARDING.md

## 1. Status
Active - Execution Sub-File

---

## 2. Purpose
Define how a new client is onboarded after agreement.

Ensures:

- clarity from day one
- aligned expectations
- smooth start

---

## 3. Core Principle

Start clear, stay clear.

---

## 4. System Rules

- Confirm scope before starting
- Keep onboarding simple
- Do not overwhelm the client
- Set expectations early

---

## 5. Execution Steps

1. Confirm agreement (what is being delivered)
2. Send welcome message
3. Outline next steps
4. Request any required information
5. Confirm timeline

---

## 6. Example

"Great to get started - here's what happens next..."

---

## 7. Output Requirement

Client must know:

- what will happen
- what is needed from them
- what to expect

---

## 8. Failure Conditions

- unclear start
- missing information
- confusion on scope

---

## 9. End State

Client feels confident and clear from the start.
