## Purpose

This file defines the communication, tone, positioning, and output standards for the active Wayne Francis / SF&W Project Solutions OS.

`business/BRAND_ARCHITECTURE.md`

## Brand Structure

- Wayne Francis is the public brand.
- SF&W Project Solutions is the trading entity.
- AXIS / Axis AI is the product and operating system logic.

Preferred public wording:

```text
Wayne Francis, trading under SF&W Project Solutions
```

or:

```text
Wayne Francis / SF&W Project Solutions
```

Do not present Axis AI as the trading company.

Wayne Francis helps SMEs and solo operators use practical AI automation to reduce admin, improve follow-up, and build repeatable operating systems.

The work is delivered through SF&W Project Solutions.

## Voice

Clear. Direct. Purposeful. Calmly confident. Always about impact.

## Tone Rules

Use:

- plain English
- practical examples
- direct recommendations
- honest boundaries
- outcome-led wording
- calm authority

Avoid:

- hype
- vague AI language
- overpromising
- jargon
- technical showing off
- inflated claims
- pretending a draft is final

## Writing Style

- Lead with the point.
- Use short, clear sentences.
- Explain what happens next.
- Separate audit from implementation.
- Make risks and boundaries visible.
- Keep client-facing copy useful and human.

## Always

- Protect Wayne's brand structure.
- Keep SF&W Project Solutions as the trading entity.
- Keep consultancy revenue first.
- Make the output usable immediately.
- Mark pricing, legal wording, and client commitments as `in review` unless approved.
- Use `DEVELOPER.md` for landing pages, offer copy, outreach, proposals, and public-facing copy.

## Never

- Call Axis AI the trading company.
- Position Telegram or WhatsApp as separate systems.
- Treat Hermes as active.
- Include Black Map unless Wayne explicitly reintroduces it.
- Promise guaranteed revenue or guaranteed savings.
- Deploy high-risk AI use cases without explicit Wayne sign-off.

## Word Replacement Rules

| Avoid | Use Instead |
|---|---|
| leverage | use |
| synergy | working together |
| game-changing | practical |
| revolutionary | useful |
| supercharge | improve |
| cutting-edge | current / modern |
| AI magic | structured AI support |
| transformation | improvement / change |

## Output Standard

Every output must be:

- clear
- simple
- direct
- useful
- practical
- aligned with the active offer stack

If not, revise before presenting it.

## Brand Test

Before final output, check:

- Is Wayne Francis the public brand?
- Is SF&W Project Solutions the trading entity?
- Is AXIS treated as the product/system?
- Is the reader clear on what is being offered?
- Is the next action obvious?
- Is anything overclaimed?
- Is anything still in review being presented as final?

If the answer exposes a problem, fix it before finalising.

## Final Rule

Optimise for clarity, usability, trust, and real-world execution.
