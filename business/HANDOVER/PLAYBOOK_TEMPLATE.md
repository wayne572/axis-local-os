## Common questions

**Q: What if the system drafts something that doesn't sound like us?**
Update BRAND.md with the correction. The system reads BRAND.md every time it generates content — improvements stick.

**Q: What if a tool stops working?**
Tell the system: "I think [tool] has stopped working." The system will diagnose. If it's a real outage or major issue, contact Wayne.

**Q: What if we want a new workflow added?**
Type the request to the system. It will route to the Workflow Agent and propose a build. Wayne reviews larger workflow additions before they go live.

**Q: What if a team member needs access?**
Add them to the tool accounts directly (Airtable, Gmail, Slack). The Axis OS folder itself is on your machine — anyone with access to that folder can run the system.

**Q: What if we want to change our brand voice?**
Edit BRAND.md directly, or run the Onboarding Agent again — it will overwrite BRAND.md with new answers.

**Q: What if Wayne disappears tomorrow?**
You still own everything. The agents are markdown files you can read. The tools are in your name. Nothing is hidden. The only thing you'd lose is ongoing optimisation — the system itself runs without him.

## What to do if something breaks

1. **Don't panic.** Nothing in Axis OS v2 is destructive. If a workflow misfires, you can re-run it.
2. **Check the tool first.** 80% of "the system is broken" turns out to be a tool issue (Zap turned off, Airtable token expired, Gmail logged out).
3. **Tell the system what you're seeing.** Be specific. Example: "The follow-up email draft didn't appear after the Typeform was submitted."
4. **If it's still broken after 10 minutes, message Wayne** — wayne@waynefrancis.co.uk.
