## What Makes Axis OS v2 Different — The Customer-Triggered Handover

When a customer buys Axis OS v2, they don't wait for a consultant to set them up. They run the setup themselves, in conversation with the system.

**They type one command:**
> `Begin Axis OS v2 setup`

**The Handover Agent then runs the entire deployment:**

1. **Welcome & scope** — explains what's about to happen, confirms the customer is ready
2. **Onboarding** — asks structured questions, writes the customer's brand, offers, content themes, and memory
3. **Diagnose & prioritise** — identifies the top 3 bottlenecks and the 2 workflows to build first
4. **Recommend the stack** — picks pre-tested tools within budget, deduplicates costs, respects off-limits
5. **Guided setup** — walks them through every tool connection, one instruction at a time
6. **End-to-end test** — runs a real test on the live system before declaring it done
7. **Team playbook** — generates a customer-specific guide their team can use day one

**What this changes for the seller:**
- 1 hour of your time per deployment, not 14 days
- The customer sees value within the first 90 minutes
- The system tests itself before sign-off — you never ship a broken deployment
- Capacity goes from 2 deployments per month to 10–15

**What this changes for the buyer:**
- They're in control from minute one
- They build the system in their business, not in your office
- They own everything immediately
- If they pause, the system resumes cleanly when they're back

**The pitch line:**
> *"You buy it on Monday. You're running on Friday. You did the setup yourself, in your business, in conversations with the system. No consultant. No installation. Yours from day one."*

**ICP:** 1–15 staff, £100k–£2m revenue, owner-led.

**Pain Points**
- Chasing clients for documents and signatures
- No follow-up on quotes or proposals
- LinkedIn dead or inconsistent
- Referral network not worked
- Manual client onboarding

**Agent Mapping**

|---|---|

| Inconsistent LinkedIn | linkedin |

**Use Cases**
1. Proposal follow-up system — auto-draft at day 3, 7, 14
2. LinkedIn content engine — weekly batch, reviewed before posting
3. Client onboarding pack — standard welcome, document checklist, kickoff agenda
4. Referral outreach sequence — keep top referrers warm
5. Monthly pipeline review — flag stalled prospects in one session

**Sprint Entry:** Lead and proposal pipeline + first LinkedIn batch.

**ROI Signal**

- LinkedIn consistency surfaces inbound enquiries within 4–6 weeks

**Sales Hook:** "How many proposals have you sent in the last 90 days that you never heard back from?"

## Where Axis OS v2 Is NOT a Fit

| Scenario | Why |
|---|---|
| Wants full automation, no human oversight | Axis OS v2 supports humans, doesn't replace judgement |
| No existing leads or audience | Amplifies what exists, can't create demand from nothing |
| Won't engage with the system | Owner must review and approve outputs |
| Needs deep legacy CRM integration | v2 uses Airtable; bespoke integration is scoped work |
| Wants a marketing agency on retainer | v2 builds the system, doesn't run campaigns |
| Needs regulated advice | v2 drafts content; doesn't produce FCA/legal sign-off |
| Wants it done for them entirely | Some involvement is always required |

1. What's falling through the cracks right now?
2. Do you have leads or clients coming in — just not being followed up properly?
