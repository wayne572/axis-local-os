# 05 - User Guide

Status: active draft
Purpose: Explain how to use AXIS day to day.

## Starting AXIS

Open the Claude project or folder connected to:

`D:\Axis AI - ChatGPT OS\AXIS_OS_CLAUDE_BUILD`

Start with one clear request.

If you are unsure, ask:

```text
What is the current status?
```

## Main Commands

### New Client Setup

```text
AXIS: NEW CLIENT
```

Use when creating a client OS after an audit, discovery call, or signed project.

### Solo Operator Setup

```text
AXIS: SOLO START
```

Use when testing or building a personal / solo entrepreneur OS.

### Daily Command

```text
AXIS: DAILY COMMAND
```

Use for daily priorities, follow-ups, admin, and focus.

### Weekly Review

```text
AXIS: WEEKLY REVIEW
```

Use to close loops and set next week.

### Capture

```text
AXIS: CAPTURE THIS
```

Use for messy notes, voice-note transcripts, ideas, or quick thoughts.

### Decision Support

```text
AXIS: DECISION
```

Use when choosing between options.

### Relationship Pulse

```text
Run relationship pulse
```

Use for follow-ups, intros, referrals, and warm relationship management.

## How To Ask For Work

Good request:

```text
Activate BUILD mode and create a one-page audit checklist for SME clients.
```

Good request:

```text
Review the Client OS Template and tell me what is missing before handover.
```

Good request:

```text
AXIS: CAPTURE THIS
I spoke to Sarah at a recruitment agency. She struggles with candidate follow-up and wants to talk next week.
```

## What AXIS Should Give Back

AXIS should give:
- clear output
- next action
- risks or blockers
- files changed if it edited files
- tracker update if work changed state

## How To Use It With Clients

1. Run the SME AI Automation Audit.
2. Identify workflows worth automating.
3. Use `AXIS: NEW CLIENT`.
4. Create a clean client OS copy.
5. Configure the client context.
6. Test the main workflows.
7. Handover the user guide.
8. Schedule the 30-day review.

## How To Use Solo Operator OS

1. Use `AXIS: SOLO START`.
2. Complete first-time setup.
3. Name the AI PA or keep Axis.
4. Run `AXIS: DAILY COMMAND`.
5. Use `AXIS: CAPTURE THIS` for notes and ideas.
6. Run `AXIS: WEEKLY REVIEW`.
7. Run `AXIS: 30 DAY REVIEW` after one month.

## User Rule

Do not ask AXIS vague questions if you need useful execution.

Give it a direction, a goal, or a messy note.

AXIS should turn that into structure.
