# Tammy Karikari Open Solo OS - Client Startup Guide

Status: open testing draft

## What This Helps With

This system helps Tammy test a personal AI operating system for priorities, admin, ideas, decisions, follow-ups, and weekly review.

It is built to answer:

- What matters today?
- What is still open?
- What needs following up?
- What admin should not be ignored?
- What decision needs making?
- What should be reviewed this week?

## Start Here

Use this command:

```text
AXIS: TAMMY START
```

If `AXIS: SOLO START` is used instead, the system should treat it the same way.

## Best First Step

Before using the system fully, complete:

`SETUP_QUESTIONNAIRE.md`

Then add:

1. Active tasks or projects into `TRACKERS/ACTIVE_LOOPS.md`
2. Admin into `LIFE_ADMIN/PERSONAL_ADMIN_TRACKER.md`
3. Pending decisions into `DECISIONS/DECISION_LOG.md`
4. Useful notes into `KNOWLEDGE/PERSONAL_KNOWLEDGE_BASE.md`

## What To Test First

Test these commands:

- `AXIS: DAILY COMMAND`
- `AXIS: CAPTURE THIS`
- `AXIS: DECISION`
- `AXIS: ADMIN CLEAR-DOWN`
- `AXIS: WEEKLY REVIEW`

## Feedback To Capture

After using the system, capture:

- what was useful
- what was confusing
- what felt too broad
- what felt missing
- what should be renamed
- what should be removed

## First 7 Days

Keep the test simple:

- use it once for daily priorities
- capture one messy thought or open loop
- use decision support once
- add one admin item
- run a short weekly review

The goal is not perfection. The goal is to see whether the system works in real use.
