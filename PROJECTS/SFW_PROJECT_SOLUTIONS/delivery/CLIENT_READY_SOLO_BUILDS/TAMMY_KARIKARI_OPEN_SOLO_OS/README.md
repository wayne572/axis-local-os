## What This Is

This is Tammy Karikari's open Solo Operator OS test build.

It helps her test whether a personal AI operating system can support:
- work
- business
- personal admin
- relationships
- ideas
- weekly priorities
- follow-ups
- open loops

## What Makes It Different

This is not a general chatbot.

It has:
- command centre
- active trackers
- weekly review
- relationship memory
- personal admin workflows
- solo business workflows
- decision support

## Start Command

Use:

```text
AXIS: TAMMY START
```

This command should load `START_HERE.md`, check `PERSONAL_CONTEXT.md`, and begin setup if Tammy's context is incomplete.

## Open Testing Mode

This build is intentionally open.

That means Tammy can test the functionality, try the commands, and give feedback before the system is marked as client-ready or locked.

During testing, keep notes on:

- what feels useful
- what feels confusing
- what command Tammy reaches for first
- what modules she ignores
- what should be renamed, removed, or simplified
- any missing workflow Tammy expected to see

## Important Update Boundary

This build does not automatically save onboarding answers.

During onboarding, the OS should gather Tammy's answers and produce suggested updates for the relevant files. Wayne or Tammy then confirms and saves the changes.
