# Idea To Action

Status: active
Purpose: Stop ideas from staying vague.

## Trigger

```text
AXIS: CAPTURE THIS
```

## Process

When the user captures an idea:

1. Summarise the idea in plain English.
2. Categorise it.
3. State the possible value.
4. Identify the minimum viable version.
5. Give one next action.
6. Decide whether to act, park, or archive.

## Idea Log

| Idea | Category | Value | Minimum Version | Next Action | State |
|---|---|---|---|---|---|

## State Rules

- active: worth moving now
- waiting: needs input or timing
- in review: needs a decision
- locked: turned into a committed project
- archived: not useful now
