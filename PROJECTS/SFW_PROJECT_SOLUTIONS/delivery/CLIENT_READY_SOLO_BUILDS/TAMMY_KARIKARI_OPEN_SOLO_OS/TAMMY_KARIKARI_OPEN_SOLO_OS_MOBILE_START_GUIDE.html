Tammy Karikari Open Solo OS

A mobile-friendly start guide for testing the Solo Operator OS functionality.

What You Have Received

This is an open test build of Tammy's Solo Operator OS. It is designed to help test daily priorities, admin, open loops, ideas, decisions, follow-ups, and weekly review.

This version is intentionally open. The goal is to test what works before anything is locked.

How To Start

  1. Download the zip file.
  2. Unzip it.
  3. Keep the whole folder together.
  4. Open START_HERE.md.
  5. Use the start command in Claude.

Start Command

Use:

AXIS: TAMMY START

If you use AXIS: SOLO START, the system should treat it the same way.

First Setup

The first setup should capture:

  • what Tammy wants help organising
  • what usually gets dropped or delayed
  • what a good week looks like
  • which commands feel useful
  • what feels confusing or missing
