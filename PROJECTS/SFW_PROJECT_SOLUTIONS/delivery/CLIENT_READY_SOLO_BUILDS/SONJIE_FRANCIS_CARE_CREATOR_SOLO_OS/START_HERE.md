# Start Here - Sonjie Francis Solo OS

Status: active
Purpose: First page to open when running Sonjie's care + creator Solo OS.

## How To Start

Use this trigger:

```text
AXIS: SOLO START
```

Claude should then check `PERSONAL_CONTEXT.md`.

If setup is incomplete, run first-time setup.

If setup is complete, ask:

```text
Do you want Daily Command Centre, Weekly Review, Care Rota Builder, TikTok Content Review, Capture, Decision Support, or Admin Clear-Down?
```

## Main Commands

```text
AXIS: DAILY COMMAND
```

Run the daily command centre.

```text
AXIS: WEEKLY REVIEW
```

Run the weekly review.

```text
AXIS: CAPTURE THIS
```

Turn a messy note, voice note, idea, or message into structured action.

```text
AXIS: IDEA TO MEMORY
```

Turn an idea into a structured plan, action, review point, and useful memory update.

```text
AXIS: DECISION
```

Help the user make a decision using options, risks, trade-offs, and next action.

```text
AXIS: ADMIN CLEAR-DOWN
```

List and prioritise personal admin.

```text
AXIS: CARE ROTA BUILDER
```

Turn rota, shift, coverage, or workplace admin problems into a practical plan or template.

```text
AXIS: TIKTOK REVIEW
```

Review content direction, next posts, audience fit, and monetisation readiness.

```text
AXIS: CREATOR PLAN
```

Build a simple content plan around consistency, clarity, audience, and realistic monetisation steps.

```text
AXIS: 30 DAY REVIEW
```

Review whether the OS is improving clarity, follow-through, admin, content direction, and practical workplace support.

## Operating Standard

Every session should create at least one of:

- a decision
- a next action
- an updated tracker
- a clearer plan
- a useful draft
- a closed loop
- a memory update only when something durable was learned
