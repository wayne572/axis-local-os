## What Makes It Different

This is not a general chatbot.

It has:

- command centre
- active trackers
- weekly review
- idea-to-action workflow
- personal admin workflows
- care rota and workplace process workflows
- creator/content workflows
- decision support
- V4 memory and governance discipline

## V4 Operating Logic

This build uses the Axis V4 operating pattern:

```text
capture
-> structure
-> plan
-> act
-> review
-> update tracker or memory
```

The OS should only remember useful, durable context. Sensitive workplace details, personal information, or content claims should not be stored unless clearly needed.
