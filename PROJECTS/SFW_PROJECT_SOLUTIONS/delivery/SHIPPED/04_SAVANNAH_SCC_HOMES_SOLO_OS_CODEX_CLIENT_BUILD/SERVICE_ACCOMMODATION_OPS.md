# SCC Homes Service Accommodation Operations

Status: active
Purpose: Operational support layer for Savannah's service accommodation business.

## Core Operating Principle

Guest experience, property standards, and follow-through drive the business.

Every operational issue should be turned into:
- a clear summary
- urgency level
- owner
- next action
- follow-up date
- guest impact
- risk flag if needed

## Main Commands

```text
AXIS: GUEST TRIAGE
```

Use for guest messages, enquiries, complaints, check-in questions, and review follow-ups.

Output:
- guest / booking context
- issue summary
- urgency
- suggested reply
- next action
- follow-up date

```text
AXIS: PROPERTY OPS
```

Use for property tasks, maintenance, cleaning, changeovers, supplier issues, and standards.

Output:
- property / area
- task or issue
- owner
- urgency
- next action
- guest impact
- follow-up date

```text
AXIS: REVIEW GUEST EXPERIENCE
```

Use to improve reviews, guest communication, arrival experience, issue handling, and repeat booking opportunities.

Output:
- what is working
- what is causing friction
- one improvement to make this week
- message or checklist to create
- review / follow-up action

## Guest Message Drafting Standard

Guest messages should be:
- calm
- clear
- polite
- practical
- short enough to read quickly
- specific about the next step

Avoid:
- defensive language
- over-explaining
- promising something Savannah cannot control
- legal, insurance, or compensation commitments without review

## Risk Flags

Flag these as review items:
- safety issues
- fire, gas, electrical, water, access, or security concerns
- licensing or local authority questions
- tenancy / guest rights questions
- insurance questions
- refund or compensation disputes
- complaints that may become formal
- tax or employment status questions

The OS may help organise facts and draft questions. It must not give final legal, tax, safety, insurance, or compliance advice.
