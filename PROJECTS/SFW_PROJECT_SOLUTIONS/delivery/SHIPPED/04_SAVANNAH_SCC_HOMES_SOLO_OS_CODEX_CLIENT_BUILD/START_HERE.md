# Start Here - Savannah / SCC Homes Solo Operator OS

Status: active
Purpose: First page to open when running Savannah's Solo Operator OS in Codex.

## How To Start

Use this trigger:

```text
AXIS: SOLO START
```

Codex should then check `PERSONAL_CONTEXT.md`.

If setup is incomplete, run first-time setup.

If setup is complete, ask:

```text
Do you want Daily Command Centre, Guest / Booking Triage, Property Ops, Weekly Review, Capture, Decision Support, or Admin Clear-Down?
```

## Main Commands

```text
AXIS: DAILY COMMAND
```

Run the daily command centre for SCC Homes.

```text
AXIS: GUEST TRIAGE
```

Sort guest enquiries, replies, complaints, check-in questions, review issues, and follow-ups.

```text
AXIS: PROPERTY OPS
```

Review property tasks, cleaning, maintenance, supplier actions, changeovers, and operational risks.

```text
AXIS: WEEKLY REVIEW
```

Run the weekly review.

```text
AXIS: CAPTURE THIS
```

Turn a messy note, voice note, idea, guest issue, property task, or message into structured action.

```text
AXIS: DECISION
```

Help Savannah make a decision using options, risks, trade-offs, and next action.

```text
AXIS: ADMIN CLEAR-DOWN
```

List and prioritise business or personal admin.

```text
AXIS: BUSINESS BUILDER
```

Focus on bookings, revenue, partnerships, direct booking opportunities, content, guest experience, and growth actions.

```text
AXIS: 30 DAY REVIEW
```

Review whether the OS is improving clarity, follow-through, admin, business progress, guest experience, and relationships.

## Operating Standard

Every session should create at least one of:
- a decision
- a next action
- an updated tracker
- a clearer plan
- a useful draft
- a closed loop
