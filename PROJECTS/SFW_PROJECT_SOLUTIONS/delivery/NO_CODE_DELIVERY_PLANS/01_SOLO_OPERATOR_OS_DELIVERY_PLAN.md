# Solo Operator OS - No-Code Delivery Plan

Status: active draft
Price: GBP 3,995 one-time, in review
Owner / brand: Wayne Francis
Trading entity: SF&W Project Solutions
Purpose: Deliver Solo Operator OS without programming.

## Who This Is For

Solo consultants, freelancers, creators, expert practitioners, and busy solo operators who need one structured system for work, admin, ideas, relationships, and follow-up.

## Client Outcome

The client receives a personal operating system they can run with Claude.

It helps them:
- plan the day
- review the week
- track open loops
- manage personal admin
- manage business actions
- capture ideas
- remember follow-ups
- make clearer decisions

## What You Need

No-code tools:
- Claude project or folder
- Solo Operator OS template folder
- setup questionnaire
- call notes
- email
- optional spreadsheet or Notion page

Source template:

`PROJECTS/SFW_PROJECT_SOLUTIONS/delivery/SOLO_OPERATOR_OS_TEMPLATE`

## Delivery Timeline

Recommended:
5-7 working days.

## Stage 1 - Fit Call

Time:
20-30 minutes.

Goal:
Confirm the client needs a personal OS, not just advice or coaching.

Ask:
- What are you running on your own?
- What keeps getting dropped?
- What tools do you currently use?
- What does a good week look like?
- Do you want one operating layer or just templates?

Decision:
- fit
- not a fit
- needs SME Client OS instead

## Stage 2 - Setup Intake

Use:

`SOLO_OPERATOR_OS_TEMPLATE/SETUP_QUESTIONNAIRE.md`

Capture:
- name
- preferred AI PA name
- main life/work areas
- current priorities
- repeated admin
- business goals
- relationship follow-ups
- weekly rhythm

No-code action:
Fill in the questionnaire during a call or from a form/email reply.

## Stage 3 - Create The Client Copy

No-code action:

1. Copy the whole `SOLO_OPERATOR_OS_TEMPLATE` folder.
2. Rename it:

```text
SOLO_OPERATOR_OS_[CLIENT_NAME]
```

3. Fill in:

`PERSONAL_CONTEXT.md`

4. Mark:

```text
Status: in review
```

Do not edit the master template for one client.

## Stage 4 - Configure The Core Files

Update:
- `PERSONAL_CONTEXT.md`
- `TRACKERS/ACTIVE_LOOPS.md`
- `LIFE_ADMIN/PERSONAL_ADMIN_TRACKER.md`
- `BUSINESS/SOLO_BUSINESS_TRACKER.md`
- `RELATIONSHIPS/RELATIONSHIP_MEMORY.md`
- `KNOWLEDGE/PERSONAL_KNOWLEDGE_BASE.md`

No-code method:
Use Claude to turn the intake notes into each file.

Prompt:

```text
Using this setup questionnaire, fill the Solo Operator OS context files. Keep everything practical, plain English, and mark uncertain items as in review.
```

## Stage 5 - First Run With Client

Run:

```text
AXIS: SOLO START
```

Then test:
- `AXIS: DAILY COMMAND`
- `AXIS: CAPTURE THIS`
- `AXIS: DECISION`
- `AXIS: ADMIN CLEAR-DOWN`
- `AXIS: WEEKLY REVIEW`

Goal:
Show the client how to use it without needing Wayne every time.

## Stage 6 - Handover

Give the client:
- the folder
- `START_HERE.md`
- main commands
- what to use daily
- what to use weekly
- what not to use it for

Explain:
The system supports organisation, planning, drafting, and decision clarity. It does not replace legal, financial, medical, tax, therapeutic, safeguarding, or regulated advice.

## Stage 7 - 30-Day Review

Schedule at setup.

Use:

`ROUTINES/30_DAY_REVIEW.md`

Review:
- what improved
- what is still messy
- which commands are used
- which modules are ignored
- what to simplify
- whether to add Telegram/WhatsApp capture later
