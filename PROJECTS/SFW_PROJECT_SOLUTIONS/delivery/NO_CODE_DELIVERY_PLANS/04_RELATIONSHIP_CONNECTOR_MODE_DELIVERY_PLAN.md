## Delivery Rhythm

Quarterly.

Suggested structure:
- Month 1: review and improve
- Month 2: light check-in
- Month 3: quarterly review and next-quarter plan

## Stage 1 - Quarterly Intake

Ask:
- What changed since the last review?
- Which relationships matter this quarter?
- Who needs a follow-up?
- Which opportunities are warm?
- Where is the system drifting?
- What new workflow pain has appeared?

## Stage 2 - Relationship Review

Update or review:
- relationship memory
- intro tracker
- follow-up list
- warm opportunities
- consent status for introductions

No-code rule:
Do not make or suggest introductions without consent.

## Stage 3 - Operations Drift Check

Review:
- active workstreams
- stale tasks
- workflows not being used
- trackers that are too heavy
- missing templates
- new repeated admin

Prompt:

```text
Review this client's current OS and identify where the system is drifting, what should be simplified, and what should be improved next quarter.
```

## Stage 4 - Quarterly Action Plan

Create:
- top 3 relationship actions
- top 3 workflow improvements
- follow-up list
- decisions needed
- next-quarter priorities

## Stage 5 - Optional Light Check-In

Mid-quarter, ask:
- What has moved?
- What is stuck?
- Who needs follow-up?
- Has anything changed in the business?

Keep this light.

## Stage 6 - End-Quarter Review

Review:
- follow-ups completed
- introductions made
- opportunities created
- workflows improved
- issues carried forward
- next quarter plan

## Deliverables

Each quarter, client receives:
- relationship pulse
- intro and follow-up tracker update
- workflow drift report
- next-quarter action plan
- updated OS recommendations

## What Is Not Included

Not included unless agreed:
- full rebuild
- new Client OS build
- custom integrations
- direct sales activity
- automated outreach
- CRM migration
- legal/compliance advice

## Completion Criteria

Quarter is complete when:
- quarterly review delivered
- next actions agreed
- trackers updated
- next review date scheduled
