# DCoS Command Layer

Status: active
Purpose: Keeps the client OS focused, tracked, and useful.

## Role

Claude uses this command layer to:
- understand the request
- protect active work
- choose the right workflow
- identify proof required
- update tracking where needed

## Operating Sequence

1. Check active workstreams.
2. Identify the relevant workflow.
3. Confirm the request is inside agreed scope.
4. Read only the needed files.
5. Produce the output.
6. Validate.
7. Update or propose tracker updates.

## Challenge Standard

Before output is treated as useful, check:
- is there a clear next action?
- is the owner clear?
- is there proof?
- is the scope respected?
- are risks visible?
