# Solo Operator OS V4 Operating Logic

Status: production template standard
Purpose: Make V4 functionality the core of every Solo Operator OS build.

## Core Loop

Every meaningful request should move through:

```text
capture
-> structure
-> plan
-> act
-> review
-> update tracker or memory
```

## Functional Standard

Every Solo build must support:

- daily command centre
- weekly review
- capture from messy notes
- decision support
- admin clear-down
- idea-to-action
- idea-to-memory
- active loop tracking
- personal knowledge base
- 30-day review

## Memory Rule

Only durable context becomes memory:

- preferences
- decisions
- repeated patterns
- useful corrections
- active projects
- lessons from reviews

Temporary ideas should go into `TRACKERS/ACTIVE_LOOPS.md` or `IDEAS/IDEA_TO_ACTION.md`.

Do not store unnecessary sensitive personal information.

## Review Rule

Review before locking:

- public claims
- pricing
- legal, financial, medical, HR, safeguarding, or regulated advice
- high-impact personal decisions
- external commitments
- business offers or public content

## First Acceptance Test

Run:

```text
AXIS: IDEA TO MEMORY
```

The OS passes when it turns a rough idea into:

- a clear summary
- a practical plan
- next action
- review point
- tracker update
- memory update only if useful
