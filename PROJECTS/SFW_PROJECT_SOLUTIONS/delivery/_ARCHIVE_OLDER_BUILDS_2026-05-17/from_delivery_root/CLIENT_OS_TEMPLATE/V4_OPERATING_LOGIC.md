# Client OS V4 Operating Logic

Status: production template standard
Purpose: Make V4 functionality the core of every Client OS build.

## Core Loop

Every meaningful client workflow should move through:

```text
capture
-> structure
-> plan / operate
-> review
-> update tracker or memory
```

## Functional Standard

Every Client OS build must support:

- daily attention review
- weekly review
- workflow capture
- decision capture
- blocker escalation
- sales-to-delivery or workflow handoff where relevant
- usage and session tracking where agreed
- governance and data boundaries
- change control
- 30-day review

## Client Memory Rule

Only store client memory that helps delivery:

- agreed scope
- decisions
- workflow state
- team roles
- recurring process patterns
- reviewed corrections
- useful lessons

Do not store:

- unnecessary personal data
- special category data unless explicitly required and governed
- private messages outside agreed workflow scope
- unreviewed claims

## Review Rule

Mark as `in review` before treating as locked:

- client commitments
- public communications
- legal/compliance wording
- pricing or scope changes
- data handling changes
- workflow authority changes
- user access or permission changes

## First Acceptance Test

Run one real or sample client workflow through:

```text
capture
-> structure
-> plan / operate
-> review
-> update tracker or memory
```

The OS passes when the client can see:

- owner
- next action
- state
- review need
- decision or blocker
- memory update only if useful
