# Solo Operator OS Delivery Checklist

Status: active draft
Purpose: Repeatable delivery process for setting up Solo Operator OS.

## Stage 1 - Fit Check

Confirm:
- the person wants structure, not just occasional AI chat
- they are willing to use the OS at least weekly
- they understand the OS is support, not professional advice
- they have one main access route to start with

Outcome:
- fit confirmed
- not a fit
- needs a simpler version

## Stage 2 - Setup Intake

Use:

`delivery/SOLO_OPERATOR_OS_TEMPLATE/SETUP_QUESTIONNAIRE.md`

Capture:
- name
- preferred AI PA name
- main life and work areas
- current priorities
- admin risks
- business or work goals
- relationship follow-ups
- working rhythm

## Stage 3 - Folder Preparation

Create a clean copy of:

`delivery/SOLO_OPERATOR_OS_TEMPLATE`

Rename the copy using this format:

```text
SOLO_OPERATOR_OS_[CLIENT_OR_USER_NAME]
```

Do not edit the master template directly for an individual user.

## Stage 4 - Context Configuration

Update:
- `PERSONAL_CONTEXT.md`
- `TRACKERS/ACTIVE_LOOPS.md`
- `LIFE_ADMIN/PERSONAL_ADMIN_TRACKER.md`
- `BUSINESS/SOLO_BUSINESS_TRACKER.md`
- `RELATIONSHIPS/RELATIONSHIP_MEMORY.md`

Mark the configured OS as:

```text
Status: in review
```

## Stage 5 - First Run

Run:

```text
AXIS: SOLO START
```

Then test:
- `AXIS: DAILY COMMAND`
- `AXIS: CAPTURE THIS`
- `AXIS: DECISION`
- `AXIS: ADMIN CLEAR-DOWN`

Confirm the OS:
- reads the user's context
- routes to the right module
- gives a useful next action
- updates or recommends tracker updates
- avoids high-risk advice

## Stage 6 - Handover

Show the user:
- where to start
- the main commands
- where personal context lives
- how to capture ideas
- how to run daily and weekly reviews
- what the OS can and cannot do

## Stage 6A - Mobile-Friendly Shipping Guide

Create a mobile-friendly HTML guide before sending the build.

Required file:

```text
[CLIENT_OR_BUILD_NAME]_MOBILE_START_GUIDE.html
```

The guide must explain:

- what the client has received
- how to download and unzip the folder
- which file to open first
- the start command
- first setup steps
- first 7 days of use
- boundaries and safe-use notes

The guide must be:

- included inside the OS folder
- available beside the zip file
- referenced in the email draft

## Stage 7 - 30 Day Review

Schedule the review at setup.

Use:

`delivery/SOLO_OPERATOR_OS_TEMPLATE/ROUTINES/30_DAY_REVIEW.md`

Review:
- clarity
- follow-through
- admin movement
- business progress
- relationship usefulness
- modules used
- modules ignored

## Completion Rule

The OS is not locked until:
- setup has been reviewed
- the user has tested the main commands
- boundaries have been explained
- the mobile-friendly guide has been created and checked
- the 30-day review is scheduled
