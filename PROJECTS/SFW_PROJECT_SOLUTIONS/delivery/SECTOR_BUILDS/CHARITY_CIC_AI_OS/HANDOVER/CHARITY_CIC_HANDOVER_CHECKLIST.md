# Charity And CIC AI OS Handover Checklist

Status: template
Purpose: Final readiness gate before handing a charity/CIC AI OS to the organisation.

## Before Build

- [ ] Intake form completed.
- [ ] Named OS owner confirmed.
- [ ] First version scope agreed.
- [ ] Data boundaries agreed.
- [ ] Safeguarding boundary reviewed.
- [ ] Tools and access confirmed.
- [ ] Success measures agreed.

## During Build

- [ ] `CLIENT_CONTEXT` completed.
- [ ] `ACTIVE_WORKSTREAMS` completed.
- [ ] Workflow index completed.
- [ ] Priority workflows built.
- [ ] Templates and prompts added.
- [ ] Governance notes added.
- [ ] Sample records added using fictional or anonymised data.
- [ ] Out-of-scope modules removed or marked future.

## Testing

- [ ] New enquiry/referral test passed.
- [ ] Follow-up test passed.
- [ ] Funding/donor pipeline test passed, if in scope.
- [ ] Volunteer/partner memory test passed, if in scope.
- [ ] Board reporting test passed, if in scope.
- [ ] Sensitive-data boundary test passed.
- [ ] Out-of-scope request test passed.
- [ ] Weekly review test passed.

## Handover

- [ ] 30-minute walkthrough completed.
- [ ] First 7 days adoption plan provided.
- [ ] Plain-English user guide provided.
- [ ] Human review rules explained.
- [ ] Organisation owner confirms they know what not to enter.
- [ ] First weekly review date agreed.
- [ ] 30-day review date agreed.
