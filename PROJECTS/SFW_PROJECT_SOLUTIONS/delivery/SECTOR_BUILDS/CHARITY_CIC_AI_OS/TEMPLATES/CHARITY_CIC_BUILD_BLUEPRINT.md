## Core Modules

| Module | Purpose | Owner | Key Views | Main Actions |
|---|---|---|---|---|
| Command Centre | Daily priorities, overdue items, decisions, and blockers |  | Today, this week, waiting | capture, prioritise, escalate |
| Referral And Enquiry Tracker | Track new requests, status, follow-up, and outcomes |  | new, active, waiting, closed | log, update, draft reply |
| Grants And Funding Pipeline | Track grant opportunities, eligibility, deadlines, evidence, applications, reports, and renewals |  | prospect, fit check, drafting, submitted, reporting, renewal | capture, check, draft, submit review, report |
| Fundraising Campaign Tracker | Plan community fundraising, appeals, events, and supporter journeys |  | idea, planning, live, follow-up, closed | plan, launch, message, follow up |
| Donor And Sponsor Stewardship | Manage donor, sponsor, and supporter relationships |  | new, warm, active, pledged, thanked, renewal | thank, update, ask, renew |
| Impact Evidence And Funder Reporting | Capture outputs, outcomes, stories, evidence gaps, and reporting deadlines |  | evidence, stories, reports due, submitted | capture, anonymise, summarise, report |
| Income Risk And Continuation Review | Track funding gaps, dependency risks, grant endings, and continuation actions |  | risks, next 30 days, next 90 days, decisions | review, flag, prioritise |
| Social Media And Content Engine | Create content plans, image prompts, captions, comments, campaign posts, and human-review flags |  | calendar, draft, review, scheduled, published | plan, draft, review, repurpose |
| Volunteer And Partner Memory | Keep relationship notes and next actions visible |  | partners, volunteers, warm contacts | capture, follow up, brief |
| Service Delivery Tracker | Track programme work and operational delivery |  | active cases/projects, blockers | update, summarise, escalate |
| Governance And Board Reporting | Prepare board updates and decision logs |  | decisions, risks, monthly summary | summarise, flag, report |
| Templates And Prompts | Reusable messages, forms, and internal scripts |  | approved, draft, archived | draft, reuse, improve |
| Usage And Improvement Log | Track what the OS is used for and what needs improving |  | wins, issues, changes | log, review, revise |

## Safeguarding Boundary

The AI OS may help organise information, draft routine messages, and prepare summaries.

It must not make safeguarding decisions, eligibility decisions, clinical decisions, legal decisions, financial decisions, or final decisions about support access.

Sensitive outputs must be reviewed by the named organisation owner before use.
