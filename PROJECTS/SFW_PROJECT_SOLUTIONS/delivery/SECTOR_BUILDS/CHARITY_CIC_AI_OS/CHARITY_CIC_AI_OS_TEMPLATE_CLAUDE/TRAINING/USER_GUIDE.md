# Charity And CIC User Guide

Status: active
Purpose: Simple guide for organisation users.

## A Note From Wayne

Wayne Francis created this AI OS as a practical token of support for charities, CICs, and community organisations.

Wayne currently runs a CIC himself, so he understands how much work sits behind the scenes in this sector.

The OS is here to help with background pressure such as grants, fundraising, social media, follow-ups, planning, reporting, and admin.

It is not here to replace the people, judgement, care, or mission inside the organisation.

## What To Ask

Use this OS to:
- see what needs attention
- update workflows
- draft messages
- track follow-ups
- track grants and funder deadlines
- prepare grant application drafts
- plan fundraising campaigns
- follow up donors and sponsors
- create social media image prompts
- draft captions and comment replies
- plan social media content calendars
- create fundraising and volunteer recruitment posts
- prepare impact evidence and funder reports
- review income risks and continuation actions
- prepare weekly reviews
- prepare board or funder summaries
- keep decisions and actions visible

## First-Time Start

You do not need to fill in a long form before using this.

Start with:

```text
We are [organisation name]. We help people with [short description]. The first thing we need help with is [grant/social media/fundraising/admin/follow-up/etc.].
```

Rough notes are fine. The OS can help tidy them up.

You can also say:

```text
We are overwhelmed this week. Help us work out what matters most.
```

Or:

```text
We are not sure where to start. Suggest one useful thing we can do today.
```

## How To Add Notes

Paste notes in plain English.

Example:

```text
New enquiry from a local partner. They want to discuss a referral pathway next week. Follow up tomorrow and keep details non-sensitive for now.
```

## What Not To Do

Do not ask the OS to make safeguarding, legal, HR, medical, financial, eligibility, or governance decisions without human review.

Do not add sensitive personal data unless it is necessary for the agreed workflow and the organisation has approved that use.

Use anonymised notes wherever possible.

Do not submit grant applications, publish fundraising appeals, or send donor/sponsor messages without human review.

Do not ask the OS to invent numbers, outcomes, or stories. If evidence is missing, the OS should say so.

Do not publish social posts, images, comments, fundraising appeals, or impact stories without human review.

Do not use real names, faces, stories, or case details unless permission has been confirmed.

## When To Ask SF&W

Ask SF&W if:
- a workflow is broken
- scope needs changing
- a new automation is needed
- compliance or safeguarding is unclear
- the organisation wants to add a new module
