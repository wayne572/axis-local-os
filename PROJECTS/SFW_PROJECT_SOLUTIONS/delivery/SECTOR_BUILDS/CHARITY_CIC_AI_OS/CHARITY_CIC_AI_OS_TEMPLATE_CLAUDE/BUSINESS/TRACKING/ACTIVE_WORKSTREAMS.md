# Active Workstreams

Status: active
Purpose: Live tracker for client work.

## Lifecycle States

Use only:
- active
- in review
- locked
- archived

## Tracker

| Name | Type | Owner | Status | Next Action | Due Date | Proof Required | Blockers |
|---|---|---|---|---|---|---|---|
| [Workflow / task name] | [workflow / asset / admin / review] | [owner] | active | [next action] | [date] | [proof] | [blocker] |

## Rules

- Every active item needs a next action.
- Every next action needs an owner.
- Client-facing changes move to `in review` before locking.
- Locked items are signed off and should not be edited casually.
