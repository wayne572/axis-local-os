# Charity And CIC Claude AI Operating System Template

Status: active sector template
Purpose: Client-ready Claude AI OS package for charities, CICs, and social impact organisations.

## What This Is

This is the client copy template for charity and CIC builds.

It is a working operating system for a small social impact team:
- clear Claude operator instructions
- charity/CIC workflow modules
- daily and weekly command centre logic
- referral, enquiry, funding, volunteer, partner, and governance tracking
- sensitive-data and safeguarding boundaries
- handover and adoption materials

## What This Is Not

This is not the internal SF&W build.

Do not include:
- SF&W strategy files
- Wayne's internal workstreams
- source archives
- other client files
- internal pricing or sales strategy unless relevant to the signed scope
- unnecessary sensitive beneficiary, volunteer, donor, or safeguarding information

## How To Use

For each charity or CIC:

1. Copy this folder.
2. Rename it to `[ORGANISATION_NAME]_AI_OS`.
3. Complete `BUSINESS/CLIENT_CONTEXT.md`.
4. Complete `BUSINESS/TRACKING/ACTIVE_WORKSTREAMS.md`.
5. Complete the sector blueprint from `../TEMPLATES/CHARITY_CIC_BUILD_BLUEPRINT.md`.
6. Remove modules not included in the first version scope.
7. Add approved workflows to `WORKFLOWS/WORKFLOW_INDEX.md`.
8. Add only fictional or anonymised sample records.
9. Run `SHIPPING_CHECKLIST.md`.
10. Run `../HANDOVER/CHARITY_CIC_HANDOVER_CHECKLIST.md`.
11. Mark the client OS as `in review`.
12. Lock only after Wayne and the organisation owner sign off.

## Universal Trigger

Use this command to start or review setup:

```text
AXIS: NEW CHARITY OS
```

Fallback trigger:

```text
AXIS: NEW CLIENT
```
