# Client Handover Pack Template

Status: active
Purpose: Explains how SF&W creates a client OS from this template.

## Build Steps

1. Copy `CLIENT_OS_TEMPLATE`.
2. Rename it to `[CLIENT_NAME]_AI_OS`.
3. Fill in `BUSINESS/CLIENT_CONTEXT.md`.
4. Ask whether the client wants to name their AI PA, or keep the default `Axis`.
5. Populate `BUSINESS/TRACKING/ACTIVE_WORKSTREAMS.md`.
6. Select workflow modules.
7. Remove unused modules from the client copy.
8. Add client-approved templates.
9. Review compliance notes.
10. Review usage tracking notice and consent options.
11. Run `HANDOVER/HANDOVER_CHECKLIST.md`.
12. Mark the client OS `in review`.

## What To Include

Include:
- client context
- agreed workflows
- trackers
- templates
- user guide
- handover checklist
- data handling notes

## What To Exclude

Exclude:
- SF&W internal strategy
- other client examples
- source archives
- pricing strategy not in client contract
- prospecting files unless included in scope
- any unrelated venture context

## Sign-Off

The client copy can be locked when:
- Wayne has reviewed it
- client context is accurate
- assistant name is confirmed or default accepted
- workflows match agreed scope
- data handling is reviewed
- usage tracking is explained and recorded
- client understands how to use it
