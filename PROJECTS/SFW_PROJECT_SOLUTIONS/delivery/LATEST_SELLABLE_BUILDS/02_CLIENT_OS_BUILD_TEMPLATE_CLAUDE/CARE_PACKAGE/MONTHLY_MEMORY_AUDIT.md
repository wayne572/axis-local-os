# Monthly Memory Audit

Status: template
Purpose: Keep client OS memory useful, scoped, approved, and safe.

## Audit Questions

Review:

- What memory was added this month?
- Is it still true?
- Is it useful beyond the current session?
- Is it within agreed client scope?
- Is it too sensitive?
- Is it duplicated elsewhere?
- Should it be moved to a tracker instead?
- Should it be archived or removed?
- Does the client need to approve it?

## Memory Categories

| Category | Keep If | Remove Or Archive If |
|---|---|---|
| Client preferences | Repeated and useful | One-off or unclear |
| Decisions | Approved and durable | Draft or superseded |
| Workflows | Active or recurring | Replaced or abandoned |
| Relationships | Useful and appropriate | Sensitive or stale |
| Lessons | Reusable | Too vague |
