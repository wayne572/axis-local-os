# Start Here - Solo Operator OS

Status: active
Purpose: First page to open when running the Solo Operator OS.

## How To Start

Use this trigger:

```text
AXIS: SOLO START
```

Claude should then check `PERSONAL_CONTEXT.md`.

If setup is incomplete, run first-time setup.

If setup is complete, ask:

```text
Do you want Daily Command Centre, Weekly Review, Capture, Idea To Memory, Decision Support, Business Builder, or Admin Clear-Down?
```

For care package review, use:

```text
AXIS: CARE REVIEW
```

Claude should review the control register, memory audit, ROI tracker, platform change watch, and any frozen workflows.

## Main Commands

```text
AXIS: DAILY COMMAND
```

Run the daily command centre.

```text
AXIS: WEEKLY REVIEW
```

Run the weekly review.

```text
AXIS: CAPTURE THIS
```

Turn a messy note, voice note, idea, or message into structured action.

```text
AXIS: IDEA TO MEMORY
```

Turn an idea into a structured plan, review point, tracker update, and memory update only when useful.

```text
AXIS: DECISION
```

Help the user make a decision using options, risks, trade-offs, and next action.

```text
AXIS: ADMIN CLEAR-DOWN
```

List and prioritise personal admin.

```text
AXIS: BUSINESS BUILDER
```

Focus on offers, leads, content, sales, delivery, or revenue actions.

```text
AXIS: 30 DAY REVIEW
```

Review whether the OS is improving clarity, follow-through, admin, business progress, and relationships.

## Operating Standard

Every session should create at least one of:

- a decision
- a next action
- an updated tracker
- a clearer plan
- a useful draft
- a closed loop
- a reviewed memory decision
