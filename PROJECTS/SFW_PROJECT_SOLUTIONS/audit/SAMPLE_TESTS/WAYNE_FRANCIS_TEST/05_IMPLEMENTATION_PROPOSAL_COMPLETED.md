## 1. Context

Following the AI Automation Audit, we identified the following priority workflow:

Workflow:
Lead-to-proposal-to-onboarding.

Current problem:
Enquiries, follow-ups, discovery notes, proposals, and onboarding actions are scattered across several channels and manual notes.

Business impact:
Follow-ups can be missed, proposals take longer than needed, and delivery handover depends too much on memory.

Recommended solution:
Build a lightweight command centre that captures leads, tracks follow-ups, structures discovery notes, drafts proposal inputs, and prepares onboarding actions.

## 2. Proposed Project

Project name:
Lead-to-Proposal Command Centre

Purpose:
Create one clear operating flow from enquiry to proposal to client handover.

The project will:
- create a lead tracker
- create a discovery notes workflow
- connect notes to proposal drafting
- create a follow-up process
- create a client handover process
- prepare a 30-day review checkpoint

The project will not:
- replace a full CRM
- automate regulated advice
- send outbound messages without review
- integrate WhatsApp or email automatically in the first version unless agreed separately
