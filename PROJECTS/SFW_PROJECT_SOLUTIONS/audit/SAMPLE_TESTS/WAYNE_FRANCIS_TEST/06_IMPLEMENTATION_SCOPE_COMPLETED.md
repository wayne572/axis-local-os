## 1. Workflow Being Improved

Workflow name:
Lead-to-proposal-to-onboarding.

Current owner:
Wayne Francis.

Current tools:
Email, LinkedIn, WhatsApp, notes, documents.

Current problem:
Leads, notes, follow-ups, proposals, and onboarding actions are scattered.

Desired outcome:
One clear command centre for tracking opportunities from first enquiry to client handover.

## 2. In Scope

| Item | Detail | Owner |
|---|---|---|
| Lead tracker | Track source, pain point, reply, next action, status | Wayne / SF&W |
| Discovery notes | Standard call notes template | Wayne / SF&W |
| Proposal workflow | Turn notes into proposal draft | Wayne / SF&W |
| Follow-up process | Define follow-up messages and dates | Wayne / SF&W |
| Handover process | Prepare client handover once accepted | Wayne / SF&W |
| 30-day review | Review performance after use | Wayne / SF&W |

## 3. Out Of Scope

| Item | Reason |
|---|---|
| Full CRM build | Not required for first version |
| Automatic WhatsApp integration | Future channel phase |
| Automatic email sending | Requires email connector and consent controls |
| Legal or compliance advice | Outside operational automation scope |
| Full historic data clean-up | Can be scoped separately |

## 4. Inputs

Client will provide:
- sample enquiries
- sample discovery notes
- sample proposal
- current onboarding steps
- preferred follow-up timing
- any compliance or data constraints

## 5. Outputs

The build will produce:
- lead tracker
- discovery notes template
- proposal workflow
- follow-up message set
- handover email
- 30-day review checklist

## 6. Acceptance Criteria

The project is complete when:
- the workflow performs the agreed task
- the client has tested it with sample data
- handover notes are provided
- known limitations are documented
- client confirms review

## 7. Change Control

Any request outside this scope should be handled as:
- minor change included
- paid change request
- future phase
- declined
