# AI ROI Tracker

Status: active
Owner / brand: Wayne Francis
Trading entity: SF&W Project Solutions
Purpose: Track whether AI and automation work is creating measurable value for Wayne and clients.

## Why This Exists

Many businesses adopt AI without proving value.

This tracker makes AXIS practical. It connects each audit, build, workflow, and client OS to measurable outcomes.

## Core Rule

Every AI improvement should be linked to at least one measurable outcome:

- time saved
- admin reduced
- follow-up improved
- errors reduced
- faster response time
- better handover
- better visibility
- revenue movement
- cost avoided
- client satisfaction

If the outcome cannot be measured, mark it as `qualitative` and define how it will be reviewed.

## ROI Stages

| Stage | Meaning |
|---|---|
| Baseline | Current way of working before change |
| Estimate | Expected improvement before build |
| Pilot | First live test |
| Measured | Actual observed result |
| Reviewed | Wayne/client reviewed whether it is worth keeping |
| Locked | Improvement accepted as part of the operating system |

## Measurement Menu

Use simple measures first.

| Outcome | Simple Measure |
|---|---|
| Time saved | Minutes/hours saved per week |
| Faster response | Average time from enquiry to reply |
| Better follow-up | Follow-ups completed vs missed |
| Less admin | Number of manual steps removed |
| Better handover | Number of missing details reduced |
| Better visibility | Open tasks known vs unknown |
| Revenue movement | Calls booked, proposals sent, deals progressed |
| Cost avoided | Tool subscription avoided, hire deferred, rework reduced |

## SME Audit ROI Prompt

Use during an audit:

```text
For each workflow in this audit, estimate the weekly time cost, error risk, follow-up risk, and revenue impact. Then rank the workflows by likely ROI and ease of implementation.
```

## Client Review Prompt

Use after implementation:

```text
Review this client OS against the AI ROI Tracker. What changed, what evidence do we have, what is still unproven, and what should be improved next?
```

## Solo Operator ROI Prompt

Use for Solo Operator OS:

```text
Review whether Solo Operator OS is improving the user's week. Measure time recovered, tasks completed, follow-ups handled, decisions logged, and admin reduced. Separate measured results from feelings or assumptions.
```

## Evidence Rules

Good evidence:

- before/after workflow comparison
- call notes
- tracker entries
- number of follow-ups completed
- number of proposals sent
- client feedback
- screenshots where appropriate
- time estimates approved by the client

Weak evidence:

- "feels better"
- vague efficiency
- unverified claims
- assumed revenue impact

## Boundary

Do not guarantee ROI.

Track evidence, show progress, and separate confirmed results from estimates.
