# Client AI Governance Pack

Status: active draft
Owner / brand: Wayne Francis
Trading entity: SF&W Project Solutions
Purpose: Lightweight client governance pack for responsible AI OS use.

## Why This Exists

Clients need to trust that AI is being used properly.

This pack helps explain:

- what the AI OS does
- what it does not do
- what data may be used
- where human review is required
- how usage may be tracked
- how improvement feedback is handled

## Pack Files

| File | Purpose |
|---|---|
| `01_CLIENT_AI_USE_NOTICE.md` | Plain-English notice explaining how the AI OS is used |
| `02_HUMAN_REVIEW_POLICY.md` | Defines when human review is required |
| `03_DATA_AND_USAGE_BOUNDARIES.md` | Explains data boundaries and sensitive information rules |
| `04_CLIENT_CONSENT_CHECKLIST.md` | Checklist before usage tracking or analytics |
| `05_AI_RISK_REGISTER_TEMPLATE.md` | Simple register for AI-related risks and controls |

## Use

Use this pack before:

- Client OS handover
- usage tracking
- relationship memory
- automation involving personal data
- any client-facing AI workflow

## Boundary

This pack is practical governance guidance, not legal advice.

Clients with regulated, high-risk, or sensitive use cases should take independent legal, data protection, or professional advice.
