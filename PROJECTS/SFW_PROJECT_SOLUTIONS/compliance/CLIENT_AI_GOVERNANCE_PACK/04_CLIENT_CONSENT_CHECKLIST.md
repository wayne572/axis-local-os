# Client Consent Checklist

Status: client-facing draft
Purpose: Checklist before enabling usage tracking, relationship memory, or analytics.

## Before Enabling Usage Tracking

- [ ] Client understands what will be tracked.
- [ ] Client understands why it is tracked.
- [ ] Client understands how it will improve the system.
- [ ] Client knows who can see the tracking information.
- [ ] Client knows how long information will be kept.
- [ ] Client knows how to request correction or deletion.
- [ ] Sensitive personal data is excluded unless specifically agreed.
- [ ] Usage tracking is documented in the handover pack.

## Before Enabling Relationship Memory

- [ ] Purpose is clear.
- [ ] Only useful business relationship context is captured.
- [ ] Sensitive personal information is avoided.
- [ ] Warm intros require consent.
- [ ] Notes are respectful and business-relevant.
- [ ] Client understands this is not hidden monitoring.

## Before Enabling Client Analytics

- [ ] Metrics are defined.
- [ ] ROI tracking method is agreed.
- [ ] Data source is clear.
- [ ] Review rhythm is agreed.
- [ ] No individual monitoring without a clear and lawful reason.

## If Unsure

Do not enable the feature.

Ask for review first.
