# Usage Tracking Policy For Client OS

Status: in review
Purpose: SF&W policy for tracking client AI OS usage legally and transparently.

## Policy Position

SF&W may track limited usage data to:
- support clients
- fix workflow issues
- improve templates
- improve onboarding
- understand which modules create value
- prioritise product improvements

Tracking must be:
- transparent
- proportionate
- limited
- explained before handover
- recorded in the client OS

## Default Tracking Model

### Essential Operational Tracking

Used for:
- support
- troubleshooting
- delivery management
- workflow continuity

Typical data:
- module used
- date
- action type
- status
- error or issue
- improvement note

### Optional Improvement Analytics

Used for:
- improving SF&W templates
- improving client onboarding
- improving modules
- developing better support material

This should be agreed with the client.

### Public Examples / Case Studies

Never use identifiable client material publicly without explicit approval.

## What Not To Track

Do not track for general improvement:
- full private prompts
- full client documents
- customer personal data
- passwords
- credentials
- sensitive data
- employee performance without a specific lawful and agreed purpose

## Required Client Template Files

Each client OS should include:

- `COMPLIANCE/USAGE_TRACKING_NOTICE.md`
- `COMPLIANCE/CLIENT_CONSENT_AND_ANALYTICS.md`
- `BUSINESS/TRACKING/USAGE_LOG.md`

## Handover Requirement

Before handover, Wayne or SF&W must explain:

- what is tracked
- why it is tracked
- what is optional
- what is not tracked
- how the client can ask questions or opt out of optional analytics

## Improvement Rule

Use anonymised or aggregated learning wherever possible.

If the learning identifies the client, ask for explicit approval before using it outside the client engagement.

## Compliance Hold

Mark `Compliance Hold` if:
- the client is unsure
- special category data is involved
- employee monitoring may be involved
- outreach data is involved
- usage logs may contain customer personal data
