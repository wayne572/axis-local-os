# Demo Commands

Status: active draft
Purpose: Commands that can be demonstrated in videos, calls, and live walkthroughs.

## Showcase Commands

| Command | Demonstrates |
|---|---|
| `AXIS: SHOWCASE START` | Choose a demo path |
| `AXIS: VOICE NOTE TO ACTION` | Convert rambling thoughts into tasks, decisions, follow-ups, and suggested tracker updates |
| `AXIS: LIFE ADMIN RESCUE` | Prioritise bills, forms, bookings, appointments, documents, and household admin |
| `AXIS: DAILY COMMAND` | Turn today into a clear operating plan |
| `AXIS: CAPTURE THIS` | Capture messy notes, ideas, or messages |
| `AXIS: DECISION` | Compare options, trade-offs, risks, and next step |
| `AXIS: BUSINESS BUILDER` | Create offers, outreach, lead follow-up, content, and delivery actions |
| `AXIS: WEBSITE BUILD` | Turn a rough business idea into a web page and launch checklist |
| `AXIS: AUDIT REVIEW` | Convert business mess into risks, opportunities, and build scope |
| `AXIS: CLIENT OS BUILD` | Show a small business operating system |
| `AXIS: HANDOVER PACK` | Create client-ready startup guide, handover note, and 30-day review plan |
| `AXIS: AIRTABLE DATABASE DEMO` | Show how AXIS creates a safe demo database plan from an Airtable reference and fictional SME client |

## Demo Response Standard

Every command should produce:

- summary
- interpretation
- output
- next action
- suggested file or tracker update

## Important Boundary

When showing suggested tracker updates, say:

```text
Suggested update
```

Do not imply files have changed automatically unless they were actually edited.

For Airtable demos, never imply the live Airtable base has been changed. Say `demo table plan` or `suggested database structure` unless a new test base was actually created.
