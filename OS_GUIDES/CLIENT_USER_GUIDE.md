# Client User Guide

Status: client-facing draft
Audience: Client OS users
Purpose: Plain-English guide for using a client operating system after setup and handover.

## What Your Client OS Is

Your Client OS is a structured AI operating system for your business.

It is designed to help you:

- organise key business information
- manage repeatable workflows
- improve follow-up
- reduce scattered admin
- keep decisions and actions visible
- review what is working
- hand work over more clearly

It is not just a chat window. It is a working system with folders, context files, workflow notes, review points, and handover rules.

## What It Is Not

Your Client OS is not:

- a replacement for your professional judgement
- legal, financial, medical, tax, HR, or regulated advice
- a hidden monitoring system
- a guaranteed revenue tool
- a fully automated system unless that has been separately scoped

It helps structure work. You remain responsible for decisions and external communication.

## How To Start

Open your Client OS folder in Claude.

Then type:

```text
Read START_HERE.md and CLAUDE.md. Confirm what this Client OS is set up to help with.
```

If you are using a guided setup, your start command may be:

```text
AXIS: NEW CLIENT
```

## Everyday Commands

| Command | What It Does |
|---|---|
| `Status` | Shows current work, open items, and next actions |
| `Next action` | Gives one practical next step |
| `Block list` | Shows what is stuck or waiting |
| `Summarise this session` | Creates a plain-English summary of the work completed |

## Workflow Commands

| Command | What It Does |
|---|---|
| `Review this workflow` | Checks whether a workflow is clear and complete |
| `Build a workflow from this process` | Turns your notes into a repeatable workflow |
| `Create a checklist for this task` | Creates a practical checklist |
| `Draft a client email` | Drafts an email using your business context |
| `Turn this into a handover note` | Creates a handover-ready summary |
| `What is missing?` | Reviews gaps and risks |

## Example Prompts

Use natural language. For example:

```text
Here is how we currently handle new enquiries. Turn this into a clear workflow.
```

```text
Review this email before I send it to a client.
```

```text
Create a checklist for onboarding a new customer.
```

```text
Summarise today's decisions and next actions.
```

```text
What should I follow up on this week?
```

## Relationship And Follow-Up

If Relationship Connector Mode is included in your setup, you can use:

| Command | What It Does |
|---|---|
| `Run relationship pulse` | Shows who needs follow-up |
| `Capture relationship note` | Records a useful relationship note |
| `Track warm intro` | Logs a possible introduction |
| `Draft follow-up message` | Writes a follow-up message |

Relationship notes should be handled with care. Do not record sensitive personal information unless it is necessary, appropriate, and consent-aware.

## Review Rhythm

Recommended rhythm:

- daily: ask for next action or quick status
- weekly: review open actions and blockers
- monthly: review what is working and what should change
- 30 days after setup: run a full review

Useful command:

```text
Run 30 day review
```

If your Client OS includes value tracking, use:

```text
Run AI ROI review
```

If your Client OS includes usage tracking, relationship memory, or sensitive workflows, use:

```text
Run client governance check
```

## Good Use

Use the Client OS for:

- drafting
- organising
- reviewing
- planning
- workflow design
- client follow-up
- decision logs
- repeatable checklists
- handover notes

## Be Careful With

Be careful with:

- personal data
- confidential client information
- HR decisions
- regulated advice
- financial commitments
- legal wording
- public claims

If something is sensitive, ask the OS to flag risks before drafting or sending anything.

## Support

If the system gives an answer that feels wrong, unclear, or too broad, ask:

```text
Review your last answer against the Client OS purpose and give me a clearer practical version.
```

If the system seems to forget context, ask:

```text
Read the key context files again and confirm what this Client OS is for.
```

## Final Rule

The Client OS should make work clearer, easier to repeat, and easier to hand over.

If it creates confusion, pause and review the setup.
