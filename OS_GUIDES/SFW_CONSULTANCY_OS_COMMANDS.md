# SF&W Consultancy OS Commands

Status: active
Audience: Wayne Francis
Purpose: Commands for running SF&W Project Solutions as the revenue engine.

## Commercial Priority

SF&W Project Solutions comes first.

The main commercial path is:

```text
SME AI Automation Audit -> Client OS Build And Handover -> Relationship Connector Mode
```

Solo Operator OS is a separate product lane.

## Lead Mode Commands

| Command | What It Does |
|---|---|
| `Activate Lead mode` | Starts lead generation or pipeline work |
| `Find prospects for SME audit` | Creates prospect criteria or a prospect list |
| `Draft outreach message` | Creates outreach copy |
| `Create sector variant` | Adapts outreach for a sector |
| `Review outreach copy` | Checks tone, claims, and CTA |
| `Update lead tracker` | Captures lead status |

## Audit Commands

| Command | What It Does |
|---|---|
| `Run SME audit workflow` | Starts the audit process |
| `Prepare discovery call pack` | Builds call questions and notes structure |
| `Complete discovery notes` | Turns raw notes into structured notes |
| `Complete audit checklist` | Turns notes into audit findings |
| `Draft audit report` | Creates the client-facing report |
| `Review audit report` | Checks clarity, risk, and next action |

## Proposal Commands

| Command | What It Does |
|---|---|
| `Draft implementation proposal` | Creates the proposal from audit findings |
| `Draft implementation scope` | Creates the delivery scope |
| `Review proposal` | Checks offer fit, claims, pricing, and boundaries |
| `Create handover email` | Drafts the client handover message |

## Delivery Commands

| Command | What It Does |
|---|---|
| `Use Client Delivery OS` | Opens client delivery workflow |
| `Build client OS copy` | Creates a client-specific OS setup from scope |
| `Create no-code delivery plan` | Builds a no-code delivery path |
| `Prepare client handover` | Creates handover assets |
| `Run client 30 day review` | Reviews whether the system is working |
| `Run AI ROI review` | Checks whether the work has measurable value |
| `Run client governance check` | Checks consent, data, usage tracking, and human review |
| `Create client-safe version` | Prepares a safe client OS handover copy |

## Copy And Positioning

| Command | What It Does |
|---|---|
| `AXIS: DRAFT LANDING PAGE` | Creates landing page copy |
| `AXIS: REVIEW COPY` | Improves copy |
| `AXIS: POSITIONING CHECK` | Checks clarity and offer fit |
| `AXIS: FINAL COPY PASS` | Final edit before sharing |

## Rules

- Lead with the client problem, not the pitch.
- Keep pricing in review until approved.
- Do not promise guaranteed savings.
- Keep audit separate from implementation.
- Keep outputs plain English and client-ready.
