## Core Commands

| Command | What It Does |
|---|---|
| `Status` | Reports current system/workstream state |
| `Workstreams` | Lists active workstreams and next actions |
| `Block list` | Shows anything stuck or waiting |
| `Next action` | Gives one concrete next step |
| `Summarise this session` | Captures the session summary |

## Mode Commands

| Command | Use When |
|---|---|
| `Activate Orchestrate mode` | You need priorities, planning, or decisions |
| `Activate Build mode` | You want an asset, document, workflow, or template created |
| `Activate Lead mode` | You want prospects, outreach, pipeline, or sales movement |
| `Activate Learn mode` | You want a market or technology brief |
| `Activate Review mode` | You want a file, plan, offer, or output audited |

You can also just describe the task. DCoS should choose the mode.

## What Not To Do

Do not ask the system to:

- publish pricing as final until approved
- use Black Map unless explicitly reintroduced
- activate Hermes as a working layer
- treat WhatsApp or Telegram as built until that access layer exists
- make legal, financial, medical, recruitment, or regulated decisions
