## Functional Risks

### Risk 1 - Too Many Specialist Files

The new build has 87 files.

Impact:
Medium. Stronger system, but Claude needs clear entry rules to avoid reading too much.

Fix:
Use `CLAUDE.md` and `START_HERE.md` reading order. Read only relevant files per task.

### Risk 2 - Client-Specific Source Material

Some delivery source assets came from WKS-style client files.

Impact:
Low if Claude follows the folder rules. Medium if source files are copied directly.

Fix:
Use `PROJECTS/SFW_PROJECT_SOLUTIONS/client-templates/SME_DEPLOYMENT_MODULES.md`. Do not use archived source files directly.

### Risk 3 - PaaS May Distract From SME Audit

Pipeline-as-a-Service is a useful asset, but it may be too narrow as the first universal SME offer.

Impact:
Medium.

Fix:
Keep SME AI Automation Audit as the front-end offer. Treat PaaS as a secondary product.
