# START HERE — Axis OS v3

## How v3 works

Type anything. **DCoS routes it.**

DCoS is the only interface. You don't pick the agent. DCoS does.

---

## The 4 Operational Commands

| Command | What it does |
|---|---|
| `Begin Axis OS setup` | DCoS routes to handover for full client onboarding |
| `Status` | DCoS reads the tracker and reports current state |
| `Workstreams` | DCoS lists active workstreams + next actions |
| `Summarise this session` | DCoS routes to session-summary |

## Other DCoS shortcuts

- `Block list` — anything stuck / waiting
- `Next action` — single next concrete action across everything
- `Onboard new client` — direct to onboarding (skips full handover flow)

## Copy And Positioning Commands

- AXIS: DRAFT LANDING PAGE - build a page from the approved offer structure.
- AXIS: REVIEW COPY - check wording, positioning, clarity, and conversion risk.
- AXIS: POSITIONING CHECK - confirm the offer is clear and not overpromised.
- AXIS: FINAL COPY PASS - tighten copy before sharing or publishing.

For these tasks, DCoS must apply DEVELOPER.md as the copy, positioning, and landing page standard.

## Anything else

Just describe the task. DCoS picks the agent.
