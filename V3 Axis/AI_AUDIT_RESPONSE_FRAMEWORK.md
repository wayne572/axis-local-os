# AI Audit Response Framework

## When an Audit Form Arrives: Step-by-Step Analysis

### SECTION 1: DIAGNOSTIC ASSESSMENT
Extract and analyze the core problem:
- **Business Profile:** Company, size, what they do, industry context
- **Team Capacity:** Current workload, repetitive task hours/week, bandwidth constraints
- **Primary Pain Point:** The #1 task they want off their plate
- **Tech Stack:** Current tools, integration quality (0-10), tool count
- **AI Maturity:** Current AI usage, team openness to change (adoption risk)
- **Vision:** 6-month goal + 3-month success definition

### SECTION 2: GAP ANALYSIS
Identify what's missing:
- **Efficiency Gaps:** Where admin time bleeds (manually enters data, scheduling, reporting, communication, etc.)
- **Integration Gaps:** Tools that don't talk to each other
- **Knowledge Gaps:** Processes not documented or standardized
- **AI Gaps:** Opportunities for AI to handle work that's currently manual
- **Adoption Gaps:** Team readiness (resistance indicators, training needs)

### SECTION 3: SOLUTION ARCHITECTURE
What they need to solve their problem:
- **Core Components:** 
  - Data/workflow automation tools
  - AI tools for specific tasks
  - Integration layer (Zapier, API connections, custom automation)
  - Process documentation
- **Quick Wins:** 1-2 high-impact, low-effort automations (first 2 weeks)
- **Foundation Work:** Underlying systems needed (data cleanup, process standardization)
- **Scaling Work:** What happens after initial wins

### SECTION 4: IMPLEMENTATION ROADMAP
Timeline to success:
- **Week 1-2:** Quick wins + team training
- **Week 3-4:** Foundation work + integration setup
- **Month 2-3:** Refine + expand automation scope
- **Month 3:** Measure results against their success definition

### SECTION 5: EXPECTED OUTCOMES
What they should see in 3 months:
- Hours freed up per person (estimated)
- Tasks automated (specific examples)
- New capacity for growth/strategy work
- Team satisfaction/adoption metrics

### SECTION 6: DELIVERABLES & NEXT STEPS
What we'll build for them:
- Custom automation blueprint
- Tool stack recommendation
- Process documentation
- Training materials
- Success metrics dashboard

When analyzing a form, I will provide:

- Primary bottleneck
- Current tooling
- AI maturity level

## The Problem
[Their #1 pain point, impact, and what's causing it]

## Proposed Solution
[Architecture and approach tailored to their situation]

## Key Principles

✓ **Specific:** Every recommendation is tied to something they told us
✓ **Achievable:** Quick wins are real (not theoretical)
✓ **Valuable:** Focused on their stated goals, not generic solutions
✓ **Honest:** Call out adoption risk, process gaps, or challenges
✓ **Measurable:** Clear metrics for success in 3 months
