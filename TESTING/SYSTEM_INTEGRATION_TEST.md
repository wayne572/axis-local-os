# SYSTEM_INTEGRATION_TEST.md

## 1. Status
Active - System Integration Test

---

## 2. Purpose
This file defines how Axis AI v4 tests whether the main system parts work together correctly.

This is not a live user test.

This is a controlled internal test to confirm that:

- Claude understands the request
- Claude routes correctly
- Specialist OS activation works
- DCoS systems behave correctly
- validation runs before final output
- the user receives one clear response

---

## 3. Core Principle

Do not test live execution until the system flow works internally.

First prove the system works in simulation.

Then move to real-world testing.

---

## 4. What This Test Checks

- Claude conversation behaviour
- user mode handling
- Specialist OS activation
- Deal Sourcing OS routing
- ChatGPT DCoS builder role
- Claude DCoS reviewer role
- output merge behaviour
- output validation
- response assembly
- memory trigger behaviour
